## 🗄️ Database Tables Reference

### 1. Authentication & User Profile Tables

#### `users`
Core login credentials, role flag, and approval state.
* `user_id` (UUID, Primary Key): Autogenerated using `gen_random_uuid()`.
* `login_id` (VARCHAR(50), Unique, Not Null): User-chosen ID. Immutable.
* `email` (VARCHAR(255), Unique, Not Null): User's primary email. Admin-editable only.
* `password_hash` (TEXT, Not Null): Hashed user password (bcrypt).
* `is_admin` (BOOLEAN, Default FALSE): Enforced single admin constraint via unique partial index.
* `status` (VARCHAR(20), Default 'PENDING'): Status workflow values: `PENDING`, `APPROVED`, `REJECTED`.
* `requested_modules` (INT[], Default '{}'): Array of requested module IDs during registration.
* `rejected_reason` (TEXT): Feedback provided by admin when a registration is rejected.
* `last_login_at` (TIMESTAMPTZ): Tracks last successful login timestamp.

#### `user_profiles`
Separates personal data from credentials to simplify edit permissions.
* `profile_id` (UUID, Primary Key): Autogenerated.
* `user_id` (UUID, Unique, Foreign Key): Links to `users.user_id` on cascade delete.
* `full_name` (VARCHAR(150), Not Null): Admin-editable only.
* `position` (VARCHAR(100), Not Null): Admin-editable only.
* `email_display` (VARCHAR(255), Not Null): Auto-synced display email.
* `address` (TEXT): Editable by user and admin.
* `mobile_no` (VARCHAR(15)): Editable by user and admin.
* `profile_photo` (TEXT): File path or URL to profile photo. Editable by user and admin.

#### `refresh_tokens`
Tracks rotating JWT refresh tokens to support safe persistent logins.
* `token_id` (UUID, Primary Key): Autogenerated.
* `user_id` (UUID, Foreign Key): Links to `users.user_id` on cascade delete.
* `token_hash` (TEXT, Not Null): Encrypted version of the token.
* `expires_at` (TIMESTAMPTZ, Not Null): Expiry checkpoint.
* `is_revoked` (BOOLEAN, Default FALSE): Revocation state.

### 2. Authorization & Module Registry

#### `modules`
Registry of active ERP submodules.
* `module_id` (SERIAL, Primary Key).
* `module_name` (VARCHAR(50), Unique, Not Null): Supported entries: `sales`, `purchase`, `product`, `manufacture`.
* `description` (TEXT).
* `is_active` (BOOLEAN, Default TRUE).

#### `user_module_access`
Stores approved module allocations.
* `access_id` (SERIAL, Primary Key).
* `user_id` (UUID, Foreign Key): Links to `users.user_id` on cascade delete.
* `module_id` (INT, Foreign Key): Links to `modules.module_id` on cascade delete.
* `granted_by` (UUID, Foreign Key): ID of the administrator who approved the access.

### 3. Purchase Module Tables

#### `vendors`
Profiles of approved material suppliers.
* `vendor_id` (UUID, Primary Key).
* `vendor_code` (VARCHAR(50), Unique, Not Null): Code identifying the vendor (e.g. `VEND-001`).
* `name` (VARCHAR(150), Not Null).
* `contact_name` (VARCHAR(100)).
* `email` (VARCHAR(255)).
* `phone` (VARCHAR(20)).
* `address` (TEXT).
* `is_active` (BOOLEAN, Default TRUE).

#### `raw_materials`
Catalog of materials used in production.
* `material_id` (UUID, Primary Key).
* `material_code` (VARCHAR(50), Unique, Not Null): Code identifying raw items (e.g. `MAT-STEEL`).
* `name` (VARCHAR(150), Not Null).
* `unit_of_measure` (VARCHAR(20), Not Null): e.g. `pcs`, `kg`, `liters`.
* `unit_price` (NUMERIC(12,2), Default 0.00).
* `reorder_level` (NUMERIC(12,3), Default 0.000): Threshold for low stock checks.
* `current_stock` (NUMERIC(12,3), Default 0.000): Real-time physical inventory.

#### `purchase_orders`
Purchase Order header tracking status.
* `po_id` (UUID, Primary Key).
* `po_number` (VARCHAR(50), Unique, Not Null): Unique sequence identifier (e.g. `PO-2026-0001`).
* `vendor_id` (UUID, Foreign Key): Links to `vendors.vendor_id` on delete restrict.
* `status` (VARCHAR(30), Default 'DRAFT'): Check options: `DRAFT`, `PENDING_APPROVAL`, `APPROVED`, `PARTIALLY_RECEIVED`, `FULLY_RECEIVED`, `CANCELLED`.
* `total_amount` (NUMERIC(12,2), Default 0.00).
* `created_by` / `approved_by` (UUID, Foreign Keys): Link to `users.user_id`.

#### `purchase_order_items`
Purchase Order line items detail.
* `po_item_id` (UUID, Primary Key).
* `po_id` (UUID, Foreign Key): Cascade delete with parent PO header.
* `material_id` (UUID, Foreign Key): Reference to material items.
* `quantity_ordered` (NUMERIC(12,3), Check > 0).
* `quantity_received` (NUMERIC(12,3), Default 0.000).
* `unit_price` (NUMERIC(12,2)).
* `total_price` (NUMERIC(12,2), Generated Always): Automatically computed as `quantity_ordered * unit_price`.

#### `goods_receipts`
Physical incoming deliveries log.
* `receipt_id` (UUID, Primary Key).
* `receipt_number` (VARCHAR(50), Unique, Not Null): e.g. `GRN-2026-0001`.
* `po_id` (UUID, Foreign Key): Reference to parent PO.
* `received_by` (UUID, Foreign Key): Reference to receiving user.
* `received_at` (TIMESTAMPTZ, Default NOW()).
* `delivery_note_ref` (VARCHAR(100)): Vendor shipment slip.

#### `goods_receipt_items`
Quantities received per line item.
* `receipt_item_id` (UUID, Primary Key).
* `receipt_id` (UUID, Foreign Key): Cascade delete with parent GRN header.
* `material_id` (UUID, Foreign Key): Reference to material.
* `quantity_received` (NUMERIC(12,3), Check > 0).

#### `vendor_bills`
Vendor invoices generated from purchase receipts.
* `bill_id` (UUID, Primary Key).
* `bill_number` (VARCHAR(100), Unique, Not Null): Vendor invoice number.
* `status` (VARCHAR(30), Default 'PENDING_PAYMENT'): Checked values: `PENDING_PAYMENT`, `APPROVED_FOR_PAYMENT`, `PAID`, `VOID`.
* `attachment_url` (TEXT): Link to physical PDF scan.

#### `stock_ledger`
Traceability log recording all inventory changes.
* `ledger_id` (UUID, Primary Key).
* `material_id` (UUID, Foreign Key): Target item.
* `transaction_type` (VARCHAR(30)): Check values: `RECEIPT`, `ISSUE`, `ADJUSTMENT`.
* `reference_id` (UUID): Reference to transaction document (e.g. `receipt_id`).
* `quantity` (NUMERIC(12,3)): Positive for inflows, negative for outflows.
* `balance_stock` (NUMERIC(12,3)): Post-movement balance.

#### `procurement_suggestions`
Automated alerts suggesting replenishment.
* `suggestion_id` (UUID, Primary Key).
* `material_id` (UUID, Foreign Key): Cascade delete with material.
* `status` (VARCHAR(30), Default 'PENDING'): Checked values: `PENDING`, `PO_CREATED`, `IGNORED`.

## 🛡️ Database Automation Triggers

### 1. Field Mutability Constraints
Triggers in `002_triggers.sql` analyze the connection session configuration parameter `erp.is_admin`:
* If `erp.is_admin` is not explicitly set to `'true'`, updates to the following fields return an error and roll back:
  * `users.email` and `users.status`.
  * `user_profiles.full_name`, `user_profiles.position`, and `user_profiles.email_display`.
* Updates to `users.login_id` are strictly blocked for **everyone** (including admins).

### 2. Email Display Sync
* Updating `users.email` automatically propagates changes to `user_profiles.email_display` to keep records in sync.

### 3. Goods Receipt (GRN) Processing
When a row is added to `goods_receipt_items`, a trigger automatically:
1. Adds the `quantity_received` to `raw_materials.current_stock`.
2. Creates a row in `stock_ledger` summarizing the ledger movement and new balance.
3. Updates `purchase_order_items.quantity_received`.
4. Recalculates PO progress: Sets PO status to `FULLY_RECEIVED` if complete, otherwise `PARTIALLY_RECEIVED`.

### 4. Low-Stock Suggestions
* When `raw_materials.current_stock` falls below its `reorder_level` (on INSERT or UPDATE), the system automatically adds a row in `procurement_suggestions` requesting replenishment.

## 💻 Backend Integration Guide (Node.js/Express)

For general edits, perform operations normally. For administrative edits (like user approvals or profile edits), specify the admin context before executing updates:

```javascript
// Example Admin approval transaction
const client = await pool.connect();
try {
  await client.query('BEGIN');
  
  // Set session context (Crucial for bypass mutability locks)
  await client.query("SET LOCAL erp.is_admin = 'true';");
  
  // Update status
  await client.query('UPDATE users SET status = $1 WHERE user_id = $2', ['APPROVED', userId]);
  
  await client.query('COMMIT');
} catch (err) {
  await client.query('ROLLBACK');
  throw err;
} finally {
  client.release();
}
```
